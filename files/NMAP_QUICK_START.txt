💡 WHY USE SUDO?
────────────────────────────────────────────────────────────────────────

Running with sudo provides:
✓ ARP ping capability (more accurate on local networks)
✓ Raw socket access for better ICMP
✓ Full nmap functionality
✓ Better detection rates

Without sudo:
- Still works but may miss some hosts
- Uses TCP connect() instead of SYN
- No ARP ping on local networks


⚙️ NMAP DETECTION METHODS USED
────────────────────────────────────────────────────────────────────────

ICMP Methods:
  -PE  → ICMP Echo request (standard ping)
  -PP  → ICMP Timestamp request
  -PM  → ICMP Netmask request

TCP Methods:
  -PS21,22,23,25,80,135,139,443,445,3306,3389,8080
       → TCP SYN to common service ports
  -PA80,443,3389
       → TCP ACK to web/RDP ports

UDP Methods:
  -PU53,161
       → UDP to DNS and SNMP ports

Options:
  -sn  → No port scan (host discovery only)
  -n   → No DNS resolution (faster)
  -v   → Verbose output

⏱️ TIMING TEMPLATES
────────────────────────────────────────────────────────────────────────

T0 (Paranoid)   - Slowest, for extreme IDS evasion
T1 (Sneaky)     - Very slow, minimal network impact
T2 (Polite)     - Slow, reduced network load
T3 (Normal)     - Default nmap timing
T4 (Aggressive) - Fast, recommended ⭐
T5 (Insane)     - Fastest, may miss hosts or trigger IDS

Recommendations:
  - Internal networks: T4 or T5
  - Client networks:   T3 or T4
  - IDS evasion:       T1 or T2
  - CERT-IN exam:      T4
